# Промпт для "братського" код рев'ю

---

Зроби детальне код рев'ю репозиторію [REPO_URL]. Це проект мого брата, він не програміст і тільки вчиться використовувати AI для генерації коду.

Формат рев'ю:

1. **Почни з похвали** -- щиро відзнач що зроблено добре. Порівняй з типовим рівнем новачків, покажи що він вже попереду. Не будь фальшивим -- хвали тільки те, що реально заслуговує.

2. **Покажи тільки реальні баги** -- не чіпай стилістику, не пиши "можна було б краще". Тільки речі які реально ламають поведінку для юзера. Для кожного багу:
   - Покажи конкретний файл і рядок
   - Покажи код з проблемою
   - Поясни простою мовою що станеться для юзера
   - НЕ фіксь, тільки покажи -- хай сам спробує пофіксити через AI

3. **Поясни ЧОМУ ці баги виникли** -- це не його помилки, а типова поведінка AI. Покажи паттерн: AI робить те що попросиш, але не робить те що НЕ попросиш.

4. **Дай гайд з промпт-інженерингу** з 4 рівнями прогресії:
   - Рівень 1: Базовий промпт ("зроби мені X") -- покажи що саме AI пропускає при такому підході
   - Рівень 2: Промпт з вимогами до якості -- додай конкретні вимоги (DRY, тести, мобілка, consistent UX)
   - Рівень 3: Ітеративний підхід -- розбий на кроки: спочатку план, потім код по модулях, потім рев'ю, потім тести
   - Рівень 4: Фрази-підсилювачі -- конкретні речення які можна додати до будь-якого промпту для кращого результату
   - Для кожного рівня напиши КОНКРЕТНИЙ приклад промпту саме для цього проекту, не абстрактний

5. **Таблиця-підсумок** -- що зроблено, що пофіксити, що додати в наступній ітерації

Тон: дружній, підтримуючий, як старший брат. Без зверхності та менторства. Пиши українською. Не використовуй емодзі.
